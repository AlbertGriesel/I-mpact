# 🌍 I/mpact — AI Environmental Impact Tracker

A friendly, AI-powered platform for measuring, understanding, planning and
reducing personal environmental impact — built for South African and broader
African households (prepaid meters, loadshedding, generators, solar, minibus
taxis and all).

**Primary metrics:** net water consumption · net electricity consumption ·
net carbon footprint (CO₂e).

## Quick start

```bash
pip install -r requirements.txt
streamlit run app.py
```

Then open http://localhost:8501.

### Enable the AI features

The app powers four AI roles — conversational assessment, bill/receipt image
extraction, result analysis and the planning assistant — with **either**
provider (auto-detected from the key you paste in **Settings → AI connection**):

- **Free — Google Gemini** (`gemini-2.5-flash`): create a key at
  [aistudio.google.com/apikey](https://aistudio.google.com/apikey), no card
  needed (~1,500 requests/day free tier). Keys start with `AIza`.
- **Paid — Anthropic Claude** (`claude-opus-4-8`): create a key at
  [console.anthropic.com](https://console.anthropic.com/settings/keys). Keys
  start with `sk-ant-`. Highest quality; used automatically when present.

You can also set a key via the environment or secrets file instead of the UI:

```bash
set GEMINI_API_KEY=AIza...          # Windows (free)   — or ANTHROPIC_API_KEY=sk-ant-...
export GEMINI_API_KEY=AIza...       # macOS/Linux
# or: copy .streamlit\secrets.toml.example .streamlit\secrets.toml  (then edit)
```

Keys stay server-side (env/secrets are git-ignored). **Without any key the app
still works end-to-end** — questionnaire, deterministic calculator, dashboard,
goals, streaks, achievements and community pages all function; AI-specific
features show clear fallbacks.

> Privacy note: free tiers may use submitted content to improve the provider's
> models. The app warns before bill upload when the free Gemini tier is active;
> for stricter data handling use a paid key or keep bill entry manual.

## Architecture

| Layer | Files | Role |
|---|---|---|
| Shared schema | `schema.py` | ONE data structure used by both assessment methods, with validation for AI-extracted values |
| Deterministic engine | `calculations.py`, `factors.py`, `tariffs.py`, `airports.py`, `vehicles.py`, `benchmarks.py`, `comparisons.py`, `goals.py` | All footprint arithmetic — published conversion factors with source/geography metadata, confidence labels (HIGH/MEDIUM/LOW), reverse municipal tariffs, Haversine flight distances, vehicle database |
| AI layer | `ai.py` | Claude API: chat collection (validated tool calls), bill extraction (vision + structured output, user review required), analysis, planner. The LLM never does the arithmetic. |
| Persistence | `database.py`, `auth` in `views/account.py` | SQLite (`data/impact.db`): users, assessments, goals, achievements, streaks, stories, private bill uploads |
| UI | `app.py`, `views/*`, section modules (`water.py`, `electricity.py`, `transport.py`, `lifestyle.py`), `visuals.py` | Streamlit multipage app; the reactive daisy mascot and nature wallpaper are embedded web components (inline SVG/CSS/JS) so deployment stays a single `streamlit run` |

### Data priority (calculation reference doc)

measured consumption → bill-extracted values → money × municipal tariff →
behavioural estimates → national averages. Every result carries a confidence
label and the dashboard's transparency panel shows exactly which method ran.

## Notes

- Demo community members (marked "(demo)") are seeded on first run so
  Explore/Leaderboard aren't empty; their history is generated by the real
  calculation engine.
- Municipal tariff tables are approximate 2024/25 snapshots for the prototype
  (Cape Town, Johannesburg, eThekwini, Tshwane, Eskom) — results from them are
  labelled MEDIUM confidence.
- UK Government 2026 GHG conversion factors are used as clearly-labelled
  proxies where South African factors don't yet exist; the SA grid factor
  (0.906 kg CO₂e/kWh) and 218 L/person/day water benchmark are official local
  figures.
